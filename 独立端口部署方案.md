# ForTest 独立端口部署方案

## 🎯 目标

- ForTeacher 为主应用（端口 318）
- ForTest 独立端口（推荐 319）
- 开发部署时互不影响
- 清晰的项目结构

---

## 📁 仓库结构方案

### 方案 A：同一仓库，独立部署（推荐 ⭐）

```
ForTeacher/                          # 主仓库
├── index.html                       # 宣传/导航页面
├── style.css
├── README.md
├── ForTest/                        # 子模块/目录
│   ├── backend/                     # Flask 后端
│   │   ├── app.py
│   │   └── requirements.txt
│   ├── frontend/                    # React 前端
│   │   ├── src/
│   │   ├── package.json
│   │   └── build/
│   └── README.md
├── ForPaper/                       # 未来模块
└── ForClass/                       # 未来模块
```

#### 优势
- ✅ 统一代码管理（一个 Git 仓库）
- ✅ 清晰的模块化结构
- ✅ 依赖关系明确（ForTest 依赖 ForTeacher 生态）
- ✅ 适合 monorepo 架构
- ✅ 共享配置和工具

#### 劣势
- ⚠️ 仓库体积较大（包含多个项目）
- ⚠️ 需要严格的代码组织规范

---

### 方案 B：独立仓库（更彻底）

```
root/
├── ForTeacher/                      # 主仓库
│   ├── index.html
│   └── style.css
├── ForTest/                        # 独立仓库
│   ├── backend/
│   └── frontend/
├── ForPaper/                       # 独立仓库
└── ForClass/                       # 独立仓库
```

#### 优势
- ✅ 每个模块完全独立
- ✅ 独立的 CI/CD 流程
- ✅ 独立的依赖管理
- ✅ 可以独立发布版本

#### 劣势
- ⚠️ 代码分散管理
- ⚠️ 需要跨仓库协作时麻烦
- ⚠️ 缺少统一的版本控制

---

## 🎯 推荐方案：方案 A（同一仓库）

### 理由

1. **ForTest 是 ForTeacher 生态的一部分**
   - 不是一个独立的 SaaS 产品
   - 共享主题风格、设计语言
   - 用户群体相同（教师）

2. **部署独立性通过端口实现**
   - ForTeacher：318
   - ForTest：319
   - 完全独立部署

3. **代码管理层面统一**
   - 便于理解整体架构
   - 可以统一更新依赖
   - 便于新人上手

## 🔧 部署架构

### 端口分配

| 应用 | 端口 | 类型 | 状态 |
|------|------|------|------|
| ForTeacher (主） | 318 | 静态页面 | ✅ 已部署 |
| ForTest | 319 | React + Flask | ⏳ 待部署 |
| ForPaper (未来） | 320 | 待定 | ❌ 未开发 |
| ForClass (未来） | 321 | 待定 | ❌ 未开发 |

### Nginx 配置

**文件**: `/etc/nginx/sites-available/fortest`

```nginx
# ForTeacher 主应用（端口 318）
server {
    listen 318;
    server_name 118.145.238.188;

    # 静态页面
    root /root/.openclaw/workspace/ForTeacher/;
    index index.html;

    # 支持单页应用（SPA）路由
    location / {
        try_files $uri $uri/ /index.html;
    }
}

# ForTest 应用（端口 319）
server {
    listen 319;
    server_name 118.145.238.188;

    # 前端
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 后端 API
    location /api {
        proxy_pass http://127.0.0.1:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket
    location /sockjs-node {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
    }
}
```

## 🚀 迁移步骤

### 步骤 1: 创建独立的 Nginx 配置

```bash
# 创建 ForTest 配置
sudo cp /etc/nginx/sites-available/fortest /etc/nginx/sites-available/fortest-319

# 创建 ForTeacher 配置
sudo tee /etc/nginx/sites-available/forteacher > /dev/null <<'EOF'
server {
    listen 318;
    server_name 118.145.238.188;

    root /root/.openclaw/workspace/ForTeacher/;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
EOF

# 启用配置
sudo ln -sf /etc/nginx/sites-available/forteacher /etc/nginx/sites-enabled/forteacher
sudo ln -sf /etc/nginx/sites-available/fortest-319 /etc/nginx/sites-enabled/fortest-319

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

### 步骤 2: 修改 ForTest Nginx 配置

编辑 `/etc/nginx/sites-available/fortest-319`：
- 将 `listen 318;` 改为 `listen 319;`
- 保持其他配置不变

### 步骤 3: 验证部署

```bash
# 访问 ForTeacher
curl -I http://118.145.238.188:318

# 访问 ForTest
curl -I http://118.145.238.188:319

# 访问 ForTest API
curl http://118.145.238.188:319/api/topics
```

## 🛠️ 开发工作流

### ForTeacher 主页开发

```bash
cd /root/.openclaw/workspace/ForTeacher/
# 修改 index.html 和 style.css
# 完成后无需重启（静态文件）
```

### ForTest 开发

```bash
# 后端
cd /root/.openclaw/workspace/ForTeacher/ForTest/backend
/root/.openclaw/venv/bin/python3 app.py

# 前端
cd /root/.openclaw/workspace/ForTeacher/ForTest/frontend
npm start

# Nginx 反向代理（端口 319）
# 无需修改，开发服务器自动热重载
```

### 互不影响示例

```
场景 1：开发 ForTest
├→ 修改 frontend/src/App.tsx
├→ React 热重载自动更新（http://118.145.238.188:319）
└→ ForTeacher 主页不受影响（http://118.145.238.188:318）

场景 2：开发 ForTeacher 主页
├→ 修改 index.html
├→ 直接访问看到更新（http://118.145.238.188:318）
└→ ForTest 继续运行（http://118.145.238.188:319）
```
