# ForTest CORS/网络问题修复方案

## 🐛 问题分析

### 错误信息
- 前端错误：`PDF 上传失败`
- Network 错误：浏览器无法访问 `http://localhost:5000/api/topics`

### 根本原因
1. **前端运行在公网**：`https://ext-melissa-beds-presentation.trycloudflare.com`
2. **API_URL 配置为 localhost**：`http://localhost:5000`
3. **浏览器安全限制**：公网页面不能访问本地 localhost 接口（CORS + 混合内容）

### 网络架构（当前 - 错误）
```
浏览器 (公网)
  ↓ 访问
https://ext-melissa-beds-presentation.trycloudflare.com (前端)
  ↓ 调用 API
http://localhost:5000/api  ❌ 浏览器无法访问本地地址
```

---

## 🔧 解决方案

### 方案 A：后端也暴露到公网（推荐）

**架构**：
```
浏览器 (公网)
  ↓ 访问
https://ext-melissa-beds-presentation.trycloudflare.com (前端)
  ↓ 调用 API
https://xxx.trycloudflare.com (后端公网地址) ✅
  ↓ 代理到
localhost:5000
```

**实施步骤**：

#### 步骤 1: 修改后端监听地址
**文件**: `backend/app.py`
**当前**:
```python
if __name__ == '__main__':
    app.run(debug=True, port=5000)
```

**修改为**:
```python
if __name__ == '__main__':
    app.run(host='0.0.0.0', debug=True, port=5000)
```

**说明**：
- `0.0.0.0` 允许从任何 IP 访问（包括 cloudflared）
- `127.0.0.1` 只允许本地访问

#### 步骤 2: 重启后端服务
```bash
# 停止当前后端
kill 40912

# 重新启动
cd /root/.openclaw/workspace/ForTeacher/ForTest/backend
/root/.openclaw/venv/bin/python3 app.py
```

#### 步骤 3: 为后端创建公网隧道
```bash
cloudflared tunnel --url http://localhost:5000
```

**预期输出**：
```
Your quick Tunnel has been created! Visit it at:
https://xxx.trycloudflare.com
```

#### 步骤 4: 修改前端 API_URL
**文件**: `frontend/src/App.tsx`
**当前**:
```typescript
const API_URL = 'http://localhost:5000/api';
```

**修改为**（假设后端公网地址为 `https://abc.trycloudflare.com`）:
```typescript
const API_URL = 'https://abc.trycloudflare.com/api';
```

#### 步骤 5: 重启前端
```bash
# 停止前端
kill 41536

# 重新启动
cd /root/.openclaw/workspace/ForTeacher/ForTest/frontend
npm start
```

---

### 方案 B：使用同一个隧道（不推荐）

**问题**：
- Cloudflare tunnel 只能转发到一个目标端口
- 无法同时转发前端（3000）和后端（5000）

**结论**：需要为前后端分别创建隧道

## ⚠️ 注意事项

### 1. 后端公网 URL 不稳定
- 每次 `cloudflared tunnel` 都会生成不同的 URL
- 重启服务后 URL 会变

### 2. 前端需要重新配置 API_URL
- 每次后端 URL 变化都要修改前端代码
- 建议将 `API_URL` 改为环境变量：
  ```typescript
  const API_URL = process.env.REACT_APP_API_URL || 'http://localhost:5000/api';
  ```

### 3. 生产环境建议
使用 Nginx 反向代理：
```
Nginx (80/443)
  ├→ / (前端静态文件)
  └→ /api (后端 API)
```
