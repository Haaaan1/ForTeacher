# ForTest 公网部署方案

## 🌐 服务器信息

- **公网 IP**: 118.145.238.188
- **开放端口**: 318
- **访问地址**: http://118.145.238.188:318

## 📋 当前架构（使用 cloudflared）

```
当前部署：
├── 前端（localhost:3000）→ cloudflared → https://ext-melissa-beds-presentation.trycloudflare.com
└── 后端（localhost:5000）→ 本地访问（127.0.0.1）

问题：
- 前端在公网，API 配置 localhost，浏览器无法访问
```

---

## 🎯 推荐方案：Nginx 反向代理

### 架构设计

```
用户浏览器
  ↓
http://118.145.238.188:318
  ↓
Nginx (端口 318)
  ├→ /              → 前端 (localhost:3000)
  └→ /api           → 后端 (localhost:5000)
```

### 优势
- ✅ 统一入口（只需 318 端口）
- ✅ 前端 API 无 CORS 问题
- ✅ 生产级部署（比开发服务器稳定）
- ✅ 可配置 SSL（HTTPS）

---

## 🔓 实施步骤

### 步骤 1: 修改后端监听地址

**文件**: `backend/app.py`

**当前**:
```python
if __name__ == '__main__':
    app.run(debug=True, port=5000)
```

**修改为**:
```python
if __name__ == '__main__':
    app.run(host='0.0.0.0', debug=True, port=5000)
```

**说明**:
- `0.0.0.0` 允许外部访问
- `debug=False` 生产环境应关闭

---

### 步骤 2: 修改前端 API_URL

**文件**: `frontend/src/App.tsx`

**当前**:
```typescript
const API_URL = 'http://localhost:5000/api';
```

**修改为**:
```typescript
const API_URL = '/api';
```

**说明**:
- 使用相对路径 `/api`
- Nginx 会将 `/api` 转发到后端 5000 端口

---

### 步骤 3: 安装 Nginx

**命令**:
```bash
# 检查是否已安装
which nginx

# 如果未安装
sudo apt-get update
sudo apt-get install nginx -y

# 启动 Nginx
sudo systemctl start nginx
sudo systemctl enable nginx
```

---

### 步骤 4: 配置 Nginx

**配置文件**: `/etc/nginx/sites-available/fortest`

**内容**:
```nginx
server {
    listen 318;
    server_name 118.145.238.188;

    # 前端静态文件
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 后端 API
    location /api {
        proxy_pass http://127.0.0.1:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # CORS 配置（如果需要）
        add_header 'Access-Control-Allow-Origin' '*';
        add_header 'Access-Control-Allow-Methods' 'GET, POST, OPTIONS';
        add_header 'Access-Control-Allow-Headers' 'Content-Type';
    }

    # WebSocket 支持（React 开发服务器热重载）
    location /sockjs-node {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
    }
}
```

**启用配置**:
```bash
# 创建符号链接
sudo ln -s /etc/nginx/sites-available/fortest /etc/nginx/sites-enabled/

# 删除默认配置
sudo rm /etc/nginx/sites-enabled/default

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

## 📊 部署架构对比

### 方案对比

| 特性 | Cloudflared 隧道 | Nginx 反向代理 |
|------|------------------|----------------|
| 稳定性 | ⚠️ 中等 | ✅ 高 |
| 性能 | ⚠️ 通过 Cloudflare | ✅ 直连 |
| 成本 | ✅ 免费 | ✅ 免费 |
| 配置复杂度 | ⚠️ 需要维护两个隧道 | ✅ 一次性配置 |
| 生产就绪 | ❌ 否 | ✅ 是 |
| 可配置 HTTPS | ⚠️ 需要 Cloudflare SSL | ✅ Nginx SSL |
| 统一入口 | ❌ 否（前后端分开） | ✅ 是（单端口）|

## 🚀 生产环境优化建议

### 1. 使用 Gunicorn 运行后端

**当前**: Flask 开发服务器（`app.run()`）
**建议**: Gunicorn（生产级 WSGI）

**安装**:
```bash
/root/.openclaw/venv/bin/pip install gunicorn
```

**启动**:
```bash
/root/.openclaw/venv/bin/gunicorn -w 4 -b 127.0.0.1:5000 app:app
```

### 2. 构建前端生产版本

**当前**: React 开发服务器（`npm start`）
**建议**: 静态文件部署（`npm run build`）

**构建**:
```bash
cd /root/.openclaw/workspace/ForTeacher/ForTest/frontend
npm run build
```

**Nginx 配置调整**:
```nginx
# 不再 proxy_pass 3000，直接提供静态文件
location / {
    root /root/.openclaw/workspace/ForTeacher/ForTest/frontend/build;
    try_files $uri $uri/ /index.html;
}
```

### 3. 配置 HTTPS（可选）

**步骤**:
1. 申请 SSL 证书（Let's Encrypt）
2. 修改 Nginx 监听 443 端口
3. 配置 SSL 证书路径
